# Contributing

## Commit Message Convention

This project uses **Conventional Commits** in the format:

```text
type(scope): description
```

### Common types

- `feat`: new functionality
- `fix`: bug fix
- `style`: visual/formatting change without behavior changes
- `refactor`: code reorganization without changing behavior
- `docs`: documentation updates
- `chore`: general tasks, project organization, or config

### Suggested scopes for this landing page

- `header`
- `hero`
- `about`
- `features`
- `pricing`
- `footer`
- `layout`
- `responsive`
- `assets`
- `seo`
- `config`

### Good examples

- `feat(hero): add signup form`
- `style(header): adjust navigation spacing`
- `fix(responsive): correct mobile overflow`
- `docs(config): add commit convention guide`

### Writing style

- Use an imperative verb (`add`, `fix`, `update`, `refactor`).
- Keep messages short and objective.
- Prefer one language across all commits for consistency.
